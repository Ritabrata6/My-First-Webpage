  Algorithms:What are they?

 
A finite set of instructions or logic, written in order, to accomplish a certain predefined task. Algorithm is not the complete code or program, it is just the core logic required to solve a problem. Can be expressed in multiple ways, like a high level informal code(pseudocode) or a flow chart.

 

Characteristics of Algorithms :

  • An algorithm should have atleast 0 or more predefined inputs
  • An algorithm should have atleast 1 or more defined outputs
  • An algorithm should be clear and unambiguous
  • Algorithms must terminate after a number of steps
  • Should be feasible with the available resources
  • An algorithm should follow either  top-down or bottom-up approach

  Pseudocode:

  • Its one of the methods used to represent an algorithm
  • It does not follow a definite syntax
  • Cannot be compiled or executed
  • Can be read and understood by programmers who are similar to logical paradigm approaches
  • Allows the control of nested structures like WHILE,FOR, etc

 Algorithm Design Techniques:

  • Divide and Conquer approach: Its a top-down approach. The algorithms which follow divide and conquer techniques divide the original problem into a set of subproblems whose solutions can be easily found out.
  • Greedy Technique: Its used to solve the optimization problems. An optimized problem is one we are given a set of instructions, which are required to either maximize or minimize. It always makes the choice best on the best space-time complexity.
  • Dynammic Programming: Its a bottom-up approach. In here we solve all the solutions to the subproblems and then combine all of those solutions together via recursion to find out the solution for the initial core problem. 
  • Branch and Bound: It focuses basically on the fact that when a subproblem whose solutions could not be made out, is to be further divided into smaller fragments till the solutions can be found out by using the method of global optimization.
  • Randomized Algorithm: A randomized algorithm is defined as an algorithm that is allowed to access a source part of the initial problem and break it into randomized bits. These are then used in turn to influence the compilation of the initial source code.

 Complexity of Algorithms:

Its very convenient to classify algorithm based on the relative amount of time needed or space needed for the compilation of the program, as a function of the input.
  1. Time Complexity: Running time of a program as a function of the input size.

    2. Space Complexity: It is the amount of space an algorithm needs to           complete the complete compilation of a program.

Based on the two parameters , the following types are found :

  • Best case: f(n) defined by the min number of steps required by an instance of size n .
  • Worst case: f(n) defined by the max number of steps required by an instance of size n.
  • Average case: f(n) defined by the average number of steps taken by an instance of size n.
